<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → /api/photos, /api/countries | Previously public data routes now require a valid session |
| cookie → req.userId → Mongo query | userId from the JWT scopes every read/write |
| GET /file/:key | Untrusted key requesting a binary; ownership must be proven |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-IDOR-LIST | Elevation of Privilege | GET /api/photos, photo-counts | mitigate | requireAuth + `userId: req.userId` on Photo.find and aggregate $match (cast to ObjectId, Pitfall 6) |
| T-02-IDOR-FILE | Elevation of Privilege | GET /api/photos/file/:key | mitigate | Photo.findOne ownership check before streaming; return 404 not 403 (Pitfall 1) |
| T-02-UNAUTH | Elevation of Privilege | all data routes | mitigate | requireAuth mounted on both routers; isolation.test asserts 401 without cookie |
| T-02-WRITE | Tampering | POST /api/photos | mitigate | Photo.create stamps userId: req.userId; userId now required at schema level |
| T-02-RESIDUE | Information Disclosure | pre-auth userId=null data | mitigate | fresh-start migration deletes orphan docs + files (D-01); userId becomes required |
| T-02-CSRF | Tampering | cookie-auth POST /api/photos | accept | sameSite=lax (set in 02-01) blocks cross-site POST for same-origin SPA |
| T-02-CORS | Information Disclosure | credentialed fetch | mitigate | CORS credentials:true + exact origin allow-list (from 02-01); client sends credentials:'include' |
</threat_model>
