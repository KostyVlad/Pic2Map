<objective>
Build the backend authentication foundation and make signup / login / logout / me
functional end-to-end. This is the thin credential slice: after this plan, a real HTTP
client can create an account, log in (with or without "Remember me"), confirm identity via
`GET /api/auth/me`, and log out — all via an httpOnly JWT cookie, with argon2id-hashed
passwords. No client UI yet (that is plan 02-02); this plan is verifiable entirely via the
test suite and curl.

Purpose: Establishes the auth primitives (User model, jose JWT utils, requireAuth middleware,
auth routes) that every later plan in this phase depends on. Implements locked decisions
D-02 (signup/login/logout scope), D-03 (argon2id + jose httpOnly cookie), D-05 (short session
+ remember-me longer expiry).
Output: User model, jwt utils, requireAuth middleware, /api/auth routes (signup/login/logout/me),
rate limiting, config + env additions, integration tests.
</objective>

    Create server/src/routes/auth.js exporting an Express Router with four endpoints,
    implementing D-02/D-03/D-05. Define a shared cookie helper: set the `auth_token` cookie with
    httpOnly:true, sameSite:'lax', secure: config.COOKIE_SECURE, and maxAge of 30d-in-ms when
    rememberMe else 1d-in-ms (Pattern 3). Define a clear helper using res.clearCookie with the
    SAME httpOnly/sameSite/secure options (must match or the browser won't clear it).

    POST /signup: validate email format and password length >= 8; lowercase the email; hash with
    `argon2.hash(password, { type: argon2.argon2id })`; create the User; on duplicate-email
    (Mongo E11000) return 400 `{ error: 'Email already registered' }` (UI-SPEC accepts signup
    enumeration here — intentional); on success signToken + set cookie + return 201
    `{ user: { id, email } }`. Signup logs the user in immediately (no email verification — D-02).

    POST /login: look up the user by lowercased email; if not found OR
    `argon2.verify(hash, password)` is false, return 401 `{ error: 'Incorrect email or password' }`
    (identical message + status for both — no enumeration); wrap argon2.verify in try/catch
    (it throws on malformed hash). On success read `rememberMe` from the body (boolean), signToken
    with it, set cookie, return 200 `{ user: { id, email } }`.

    POST /logout: clear the cookie, return 200 `{ ok: true }`. No auth required to log out.

    GET /me: apply requireAuth; load the user by req.userId; return `{ id, email }` (404 if the
    user no longer exists). This is the ONLY auth route behind requireAuth.

    Apply express-rate-limit to /signup and /login (a limiter ~ windowMs 15min, max ~10,
    standardHeaders, message matching UI-SPEC "Too many attempts. Wait a moment before trying
    again."). Use skipSuccessfulRequests on /login so only failed attempts count.

    In app.js: import the auth router and mount it `app.use('/api/auth', authRouter)` BEFORE the
    photos/countries routes — public, NOT behind requireAuth (Pattern 4 note).

    Create server/test/auth.test.js using node --test + node:assert, following the e2e test's

    logout returns Set-Cookie clearing auth_token; /me with the login cookie returns the email;
    /me without a cookie returns 401. Assert no plaintext password is stored (the persisted
    User has passwordHash, not password).
  </action>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → /api/auth/* | Untrusted email/password/rememberMe input crosses here |
| cookie → requireAuth | Untrusted JWT presented as session proof |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-CS | Spoofing | POST /login | mitigate | express-rate-limit on /login (+/signup); argon2.verify is constant-work; generic 401 message |
| T-02-PW | Information Disclosure | password storage | mitigate | argon2id hash only; never store/log plaintext (D-03); test asserts no `password` field |
| T-02-JWT | Spoofing/Tampering | auth_token cookie | mitigate | jose HS256 signature (D-03); httpOnly + sameSite=lax; secure in prod only (Pitfall 3) |
| T-02-XSS | Information Disclosure | session cookie theft | mitigate | httpOnly cookie inaccessible to JS (D-03 — never localStorage) |
| T-02-ENUM | Information Disclosure | POST /login | mitigate | identical 401 + message for unknown-email and wrong-password (Pitfall 5) |
| T-02-CSRF | Tampering | cookie-auth POST routes | accept | sameSite=lax blocks cross-site POST; sufficient for same-origin SPA (RESEARCH Don't-Hand-Roll) |
| T-02-SC | Tampering | npm installs (jose/argon2/resend/cookie-parser/express-rate-limit) | mitigate | RESEARCH legitimacy audit (all OK) + blocking human checkpoint before continuing |
</threat_model>
