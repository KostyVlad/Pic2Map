<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → /api/auth/forgot-password | Untrusted email; must not reveal account existence |
| emailed link → /reset-password | Raw token from the user's inbox is the only proof of identity |
| server → Resend (external) | Reset link leaves the trust boundary via email |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-RST-HARVEST | Elevation of Privilege | reset token | mitigate | sha256 hash at rest; raw token only in email; 1h TTL; single-use (cleared after use) — Pattern 6 |
| T-02-RST-ENUM | Information Disclosure | POST /forgot-password | mitigate | always 200 "check your email" regardless of match (Pitfall 5) |
| T-02-RST-LEAK | Information Disclosure | reset link in URL | mitigate | token in query param not path (Pitfall 7); short TTL bounds the window |
| T-02-RST-BRUTE | Spoofing | POST /forgot-password / reset | mitigate | express-rate-limit on forgot-password; 256-bit token space makes guessing infeasible |
| T-02-RST-MAIL | Denial of Service | Resend send failure | accept | try/catch keeps the 200 contract; logged server-side; SMTP fallback adapter documented (D-06) |
| T-02-SC-RESEND | Tampering | resend package (installed in 02-01) | mitigate | covered by RESEARCH legitimacy audit + the 02-01 package checkpoint |
</threat_model>
