<objective>
Deliver the global GPS-auto-placement upload slice: a user uploads a batch from a new
map-level Upload control, the server reads GPS from each photo's EXIF (D-01), runs
point-in-polygon to auto-assign the country, stores the exact coordinates, and the UI reports
how many photos were auto-placed vs. need manual placement (D-03). Covers GEO-01, GEO-02, and
the no-GPS/no-match fallback reporting half of GEO-05.

Purpose: This is the core "magic" of Phase 3 — photos place themselves on the map by GPS.
Output: server GPS + country-lookup services, the SU_A3 server keying fix (blocking
prerequisite), an enriched upload route + response, the `useUploadGlobal` mutation,
`GlobalUploadButton`, and `GpsResultSummary`.
</objective>

## Phase Goal

**As a** PhotoMap user, **I want to** drop a batch of geotagged photos into one global Upload control, **so that** each photo lands in the correct country automatically without me picking it, and I'm told clearly which photos had no location.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → POST /api/photos | Untrusted multipart upload (files + optional countryCode) crosses here |
| photo EXIF → server | GPS coordinates inside EXIF are untrusted attacker-controllable data |
| served file route → browser | Per-user image bytes must not leak across accounts |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-GPS-INJ | Tampering | extractGps → resolveCountry | mitigate | Gate every coord through isValidGps (number, finite, lat∈[-90,90], lng∈[-180,180]) before resolveCountry/storage (RESEARCH Security Domain) |
| T-03-EXIF-LEAK | Info Disclosure | ingest pipeline / served files | mitigate | GPS read only from raw pre-ingest buffer (D-01); served files stay EXIF-stripped (ingest unchanged); coordinates returned only in JSON to the owner, never embedded in image bytes |
| T-03-IDOR-UPLOAD | Info Disclosure | POST /api/photos (global) | mitigate | Photo.create keeps `userId: req.userId`; global-upload photos scoped to uploader — no cross-user placement |
| T-03-NOCC-BYPASS | Tampering | relaxed countryCode guard | accept | Removing the required-countryCode 400 is intended (global upload); no-GPS/no-match files are gracefully skipped into noGps list — no server error, no data corruption (Pitfall 9) |
| T-03-XSS-FILENAME | Tampering | GpsResultSummary / future PinPopup | mitigate | originalFilename rendered via React (auto-escaped); no dangerouslySetInnerHTML |
| T-03-DOS | Denial of Service | global upload batch | accept | Existing multer limits (25MB/file, 50/batch) + magic-byte allowlist still apply to the same route (upload.js unchanged) |
| T-03-SC | Tampering | npm installs (exifr, @turf/boolean-point-in-polygon) | mitigate | Both [Approved] in RESEARCH Package Legitimacy Audit (verified registry, high downloads, known repos); no [ASSUMED]/[SUS]/[SLOP] packages — no blocking checkpoint required |
