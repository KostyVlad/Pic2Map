## Phase 3 Component Inventory

### 1. PhotoPinMarker (new ﻗ Leaflet DivIcon)

Purpose: render a single photo's exact GPS coordinates as a pin inside the country drill-down map.

Visual specification:
- Shape: circle, 16px diameter
- Fill: `--color-accent` (#3b82f6), solid
- Border: 2px solid `--color-surface` (#ffffff) ﻗ white ring for contrast on any tile background
- Drop shadow: `0 1px 4px rgba(0,0,0,0.3)` ﻗ same depth convention as existing Leaflet controls
- On hover: border changes to `--color-accent-dark` (#1d4ed8), 2px; cursor: pointer
- On selected/active: scale to 20px diameter, border becomes 2.5px `--color-accent-dark`
- Implemented as `L.divIcon` with `className: ''` (suppress Leaflet default white square), matching `PhotoCountBadge` implementation pattern
- `iconAnchor`: [8, 8] (center-anchored ﻗ pin is a dot, not a teardrop)
- `iconSize`: [16, 16]
- No thumbnail inside the pin ﻗ thumbnails shown in the sidebar gallery, not on map (performance and legibility)

Interaction on click:
- Opens a lightweight Leaflet popup (see PinPopup spec below) anchored to the pin
- Does NOT open the full lightbox directly from the map pin
- The sidebar remains open (it is already open ﻗ user drilled into this country)

### 2. PinPopup (new ﻗ Leaflet Popup via react-leaflet `<Popup>`)

Purpose: show thumbnail + photo name when a pin is clicked inside the country map.

Visual specification:
- Width: 180px fixed
- Background: `--color-surface` (#ffffff)
- Border: 1px solid `--color-border` (#e5e7eb)
- Border-radius: 6px
- Box-shadow: `0 2px 8px rgba(0,0,0,0.15)`
- Thumbnail: 180px wide ﺣ 120px tall, `object-fit: cover`, top section of popup
- Caption area: 8px top padding, 8px bottom padding, 8px horizontal padding
- Caption text: `text-label` (14px), weight 400, `--color-text`, truncated to 1 line with ellipsis
- No close button override ﻗ use Leaflet default popup close (ﺣ) in top-right

Interaction:
- Click thumbnail inside popup ﻗ open the existing lightbox (yet-another-react-lightbox) with this photo at index 0 in the country's photo array
- One popup open at a time (Leaflet default behavior)

### 3. ClusterMarker (new ﻗ Leaflet DivIcon via use-supercluster)

Purpose: replace multiple nearby pins with a single bubble showing the count.

Visual specification:
- Shape: circle
- Diameter tiers (by point_count):
  - 2ﻗ9 photos: 32px
  - 10ﻗ99 photos: 40px
  - 100+ photos: 48px
- Fill: `--color-accent-subtle` (rgba(59,130,246,0.15))
- Border: 2px solid `--color-accent` (#3b82f6)
- Count label: `text-label` (14px), weight 600, color `--color-accent` (#3b82f6), centered
- Font: `--font-sans` (system-ui stack)
- Drop shadow: none (clusters are less prominent than individual pins)
- On hover: fill transitions to rgba(59,130,246,0.25), border to `--color-accent-dark` (#1d4ed8); cursor: pointer
- `iconAnchor`: center of circle ([diameter/2, diameter/2])
- `iconSize`: [diameter, diameter]
- Implemented as `L.divIcon` with `className: ''`

Interaction on click:
- Zoom the map in by 2 levels centered on the cluster's coordinates (standard supercluster expand behavior via `supercluster.getClusterExpansionZoom`)
- Do NOT open a popup on cluster click

### 4. CountryPinMap (new ﻗ react-leaflet MapContainer for drill-down view)

Purpose: a map shown inside or alongside the CountrySidebar when drilling into a country ﻗ displays that country's photo pins and clusters.

Layout:
- Rendered inside CountrySidebar below the upload zone, replacing or above the PhotoGallery
- Height: 240px fixed on desktop (md+), 180px on mobile
- Width: 100% of sidebar (360px on desktop)
- Tile provider: same VITE_TILE_URL / Stadia alidade_smooth as WorldMap
- Initial bounds: fit to the country's GeoJSON bounding box (Leaflet `fitBounds`) with 20px padding
- minZoom: 3, maxZoom: 18 (allow street-level zoom for precise placement)
- Zoom control: shown (top-left)
- No attribution bar (defer to WorldMap ﻗ kept in same session)

Cluster behavior:
- `use-supercluster` with `radius: 60`, `maxZoom: 17`
- Clusters expand on zoom (click ﻗ `getClusterExpansionZoom` ﻗ flyTo)
- At maxZoom 17 all pins are individual ﻗ no cluster at this zoom

### 5. GpsResultSummary (new ﻗ inline in PhotoUploadForm)

Purpose: after upload, show how many photos were auto-located vs. how many need manual country assignment.

Position: replaces the existing `statusMessage` section in `PhotoUploadForm` for Phase 3 uploads (or renders below it).

Visual specification:
- Container: same `p-4 border-b border-border` wrapper as the existing upload zone
- Each result row: `flex items-center gap-2`, height 32px, `text-label` (14px)
- Auto-located row: accent dot (8px circle, fill `--color-accent`) + text "N photo(s) auto-placed in [Country]"
  - Text color: `--color-text` (#111827)
- No-GPS row: muted dot (8px circle, fill `--color-text-muted`) + text "N photo(s) added to [Country] (no GPS)"
  - Text color: `--color-text-muted` (#6b7280)
- Error row: `--color-destructive` (#dc2626) text, same pattern as existing error `statusMessage`
- Auto-clear: success rows clear after 4 seconds (matching existing behavior); error rows stay until dismissed

## Interaction Flows

### Flow 1: Upload with GPS photos (GEO-01, GEO-02)

1. User drops or picks photos in `PhotoUploadForm`
2. Server reads EXIF GPS, performs point-in-polygon, stores `lat`, `lng`, `countryCode`
3. Upload resolves ﻗ `GpsResultSummary` renders:
   - "3 photos auto-placed in France" (accent dot)
   - "1 photo added to France (no GPS)" (muted dot, if mixed batch)
4. `CountryPinMap` re-renders with new pins (via React Query invalidation on same `countryCode`)
5. Status clears after 4 seconds

### Flow 2: Upload without GPS (GEO-05)

1. User has opened a country and drops GPS-less photos
2. Server assigns all to the open `countryCode` (Phase 1 manual fallback)
3. GpsResultSummary renders: "2 photos added to France (no GPS)" (muted dot)
4. No pins appear on `CountryPinMap` for these photos (no coordinates)
5. Photos appear only in `PhotoGallery` (existing Phase 1 behavior)

### Flow 3: Drill into country with pins (GEO-03, GEO-04)

1. User clicks a country on WorldMap ﻗ CountrySidebar opens
2. CountryPinMap renders above PhotoGallery; bounds fit to country
3. Pins appear for photos with GPS; clusters appear where pins are dense
4. User zooms in ﻗ clusters split into individual pins
5. User clicks a pin ﻗ PinPopup appears with thumbnail
6. User clicks thumbnail in PinPopup ﻗ opens lightbox at that photo's index

### Flow 4: No photos yet in country

1. User opens a country with zero photos
2. CountryPinMap renders (empty ﻗ no pins, no clusters)
3. PhotoGallery shows empty state: "No photos yet" (existing Phase 1 copy)
4. Upload zone prompts user to add photos
