# Phase 1 — UI Design Contract
## Country Map & Per-Country Photos

> Visual and interaction contract for Phase 1 frontend.
> Generated by gsd-ui-researcher. Consumed by gsd-ui-checker, gsd-planner, gsd-executor, gsd-ui-auditor.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none — plain React components with CSS custom properties |
| Icon library | none in Phase 1 — use text labels for actions; Phase 4 may add Lucide |
| Font | system-ui stack (no web font download — keeps Phase 1 lean) |

**Rationale:** The MERN stack research lists no CSS framework. Shadcn/Tailwind would require adding postcss + tailwind config to the Vite project — unnecessary overhead for an MVP that establishes conventions. All design tokens are declared as CSS custom properties in a single `client/src/styles/tokens.css` file, imported once in `main.jsx`. This file is the single source of truth for all Phase 1 styling.

**Font stack (body and UI):**
```css
font-family: system-ui, -apple-system, BlinkMacSystemFont, 'Segoe UI', Roboto, sans-serif;
```

## Spacing Scale

Declared values (all multiples of 4):

| Token | Value | CSS Custom Property | Usage |
|-------|-------|---------------------|-------|
| xs | 4px | `--space-xs` | Icon gaps, chip padding, tight inline spacing |
| sm | 8px | `--space-sm` | Compact element spacing, input inner padding |
| md | 16px | `--space-md` | Default element spacing, card padding |
| lg | 24px | `--space-lg` | Section padding, panel header padding |
| xl | 32px | `--space-xl` | Layout gaps, gallery grid gap |
| 2xl | 48px | `--space-2xl` | Major section breaks |
| 3xl | 64px | `--space-3xl` | Page-level vertical rhythm (rarely used) |

Exceptions:
- **Touch targets on mobile:** minimum 44px height for all interactive elements (upload button, gallery thumbnails, close button). This exceeds the scale — use explicit `min-height: 44px` where needed rather than a scale token.
- **Map full-bleed:** `height: 100dvh; width: 100%` — no token, set in component CSS directly.

## Typography

Four roles, two weights. Body at 400, labels and headings at 600.

| Role | Size | Weight | Line Height | CSS Custom Property | Usage |
|------|------|--------|-------------|---------------------|-------|
| Body | 16px | 400 | 1.5 | `--text-body` | Gallery captions, upload instructions, error messages, general prose |
| Label | 14px | 600 | 1.4 | `--text-label` | Button text, photo count badge, form field labels, tab labels |
| Heading | 20px | 600 | 1.2 | `--text-heading` | Country name in panel header |
| Display | 28px | 600 | 1.1 | `--text-display` | First-run empty state headline only |

**CSS declarations:**
```css
--font-size-body:    16px;
--font-size-label:   14px;
--font-size-heading: 20px;
--font-size-display: 28px;

--font-weight-regular:  400;
--font-weight-semibold: 600;

--line-height-body:    1.5;
--line-height-label:   1.4;
--line-height-heading: 1.2;
--line-height-display: 1.1;
```

**Rules:**
- No other font sizes are permitted in Phase 1.
- Do not introduce a third weight. Bold (700) and light (300) are out of scope.
- Truncate long country names in the panel header with `text-overflow: ellipsis; white-space: nowrap; overflow: hidden` rather than wrapping.

## Color

60/30/10 split. The map tile (Stadia `alidade_smooth`) is a neutral grey-beige — the UI chrome must not compete with the map.

| Role | Hex | CSS Custom Property | Usage |
|------|-----|---------------------|-------|
| Dominant (60%) | `#f8f9fa` | `--color-bg` | App background behind the map, panel body background |
| Secondary (30%) | `#ffffff` | `--color-surface` | Country side-panel surface, upload dropzone background, lightbox toolbar background |
| Accent (10%) | `#3b82f6` | `--color-accent` | **Reserved: has-photos country polygon fill, upload CTA button, active selection ring** |
| Accent dark | `#1d4ed8` | `--color-accent-dark` | Selected country polygon border (hover/selected state) |
| Accent subtle | `rgba(59,130,246,0.15)` | `--color-accent-subtle` | Upload dropzone drag-active highlight, photo count badge background |
| Neutral border | `#e5e7eb` | `--color-border` | Panel border, gallery thumbnail border, dropzone idle border |
| Text primary | `#111827` | `--color-text` | All body copy, headings, labels |
| Text muted | `#6b7280` | `--color-text-muted` | Empty state body copy, file count secondary text |
| Destructive | `#dc2626` | `--color-destructive` | Not used in Phase 1 (no delete action). Reserved for Phase 4 delete confirmation. |
| Map overlay | `rgba(0,0,0,0.4)` | `--color-overlay` | Lightbox backdrop |

**Accent reserved for:** has-photos country polygon fill, upload CTA button background, selected country polygon border color (`--color-accent-dark`). Never used for body text, borders, or passive states.

**Map layer colors (Leaflet polygon styles — not CSS custom properties, set in JS):**

| State | fillColor | fillOpacity | color (border) | weight |
|-------|-----------|-------------|----------------|--------|
| Default — no photos | `#e5e7eb` | 0.2 | `#9ca3af` | 1 |

## Layout

### App Shell (desktop — viewport >= 768px)

```
┌─────────────────────────────────────────────────────────┐
│                    WorldMap (full-bleed)                 │
│                  height: 100dvh                         │
│                  width: 100vw                           │
│                                                         │
│                                         ┌─────────────┐ │
│                                         │  Country    │ │
│                                         │  Panel      │ │
│                                         │  width:360px│ │
│                                         │  height:100%│ │
│                                         └─────────────┘ │
└─────────────────────────────────────────────────────────┘
```

- The map is always full-bleed (`position: fixed; inset: 0`).
- The country panel overlays the map on the right (`position: fixed; top: 0; right: 0; height: 100dvh; width: 360px; z-index: 400`).
- When no country is selected, the panel is absent (not rendered, not hidden).
- Leaflet's default z-index is 400 for the tile layer. The panel must use `z-index: 500` to sit above map controls.

### App Shell (mobile — viewport < 768px)

```
┌──────────────────────┐
│   WorldMap           │
│   height: 45dvh      │
│                      │
├──────────────────────┤
│   Country Panel      │
│   height: 55dvh      │
│   overflow-y: auto   │
└──────────────────────┘
```

- On mobile, the layout splits vertically: map top 45dvh, panel bottom 55dvh.
- The panel is a scrollable column (not a modal/drawer slide-in — simpler for Phase 1).
- The map portion remains interactive above the panel divider.
- Implementation: CSS media query at 768px breakpoint toggling the layout mode.

### Country Panel Internal Layout

```
┌──────────────────────────────┐
│ [Country Name]    [✕ Close]  │  header — --space-lg padding, --text-heading
├──────────────────────────────┤
│ [Upload Area]                │  dropzone — see Upload section
├──────────────────────────────┤
│ Photo grid (thumbnails)      │  gallery — see Gallery section
│                              │
└──────────────────────────────┘
```

## Component Specifications

### Country Panel

| Property | Value |
|----------|-------|
| Width (desktop) | 360px fixed |
| Height | 100dvh (desktop) / 55dvh (mobile) |
| Background | `--color-surface` |
| Border | 1px solid `--color-border` on left edge (desktop) / top edge (mobile) |
| Box shadow | `0 0 24px rgba(0,0,0,0.12)` |
| Padding | `--space-lg` (header), `--space-md` (body) |
| Overflow | `hidden` on panel; `auto` on the gallery scroll region |
| z-index | 500 |
| Transition | none in Phase 1 — panel appears/disappears instantly |

**Panel header:**
- Country name: `--text-heading`, `--font-weight-semibold`, `--color-text`, single line with ellipsis.
- Close button: text "Close" at `--text-label`, `--color-text-muted`. Position: right edge of header row. Minimum size 44x44px touch target.

### Upload Zone (inside country panel)

| Property | Value |
|----------|-------|
| Layout | Full width of panel body, above gallery |
| Idle state | Dashed border 2px `--color-border`; background `--color-surface`; padding `--space-lg` |
| Drag-active state | Dashed border 2px `--color-accent`; background `--color-accent-subtle` |
| Height (idle) | 80px collapsed; expands to show file list on drag |
| Upload CTA button | See Copywriting section below |
| File input | Hidden `<input type="file" multiple accept="image/jpeg,image/png,image/webp,.heic,.heif">` |
| Accepted format hint | `--text-label`, `--color-text-muted`: "JPEG, PNG, WebP, HEIC — up to 25 MB each" |

**Upload button spec:**
- Background: `--color-accent` (`#3b82f6`)
- Text: `--color-surface` (`#ffffff`), `--font-weight-semibold`, `--text-label`
- Padding: `--space-sm` vertical, `--space-md` horizontal
- Border radius: 6px
- Hover: `--color-accent-dark` (`#1d4ed8`) background
- Min height: 44px (touch target)
- State — uploading: button disabled, text changes to "Uploading..." (see Copywriting)

### Photo Gallery Grid (inside country panel)

| Property | Value |
|----------|-------|
| Layout | CSS grid, `grid-template-columns: repeat(3, 1fr)` |
| Gap | `--space-xs` (4px) between thumbnails |
| Thumbnail size | 1:1 aspect ratio, fills grid column (`aspect-ratio: 1; object-fit: cover`) |
| Border radius | 4px on each thumbnail |
| Cursor | `pointer` on thumbnails |
| Hover | `opacity: 0.85` overlay |
| Scroll region | Panel body below upload zone; `overflow-y: auto` |
| Minimum thumbnail size | 80px (panel 360px - 2×16px padding - 2×4px gap) / 3 = ~105px — satisfies 44px min touch target |

**Photo count badge (on map country polygon):**
- Rendered as a Leaflet `DivIcon` positioned at the country centroid.
- Background: `--color-accent-subtle`; border: 1px solid `--color-accent`; text: `--color-accent`, `--text-label`, `--font-weight-semibold`.
- Shape: pill — `border-radius: 12px; padding: 2px 8px`.
- Show only when count >= 1. Do not render for countries with zero photos.
- Maximum displayed: if count > 99, show "99+".

### Lightbox

Library: `yet-another-react-lightbox` v3.32.0 (from RESEARCH.md).

| Property | Value |
|----------|-------|
| Backdrop | `--color-overlay` (`rgba(0,0,0,0.4)`) — library default is darker; override to 0.4 |
| Open trigger | Click on any thumbnail in the gallery |
| Navigation | Keyboard arrow keys + on-screen prev/next chevrons (library default) |
| Close | Escape key + click outside + "✕" button (library default) |
| Zoom | Disabled in Phase 1 (defer to Phase 4 polish) |
| Thumbnails strip | Disabled in Phase 1 (gallery grid serves this role) |
| Slide count indicator | Enabled (library default "N / total") |

**YARL CSS override** (import after library CSS):
```css
.yarl__backdrop { background: rgba(0, 0, 0, 0.4); }
```

## Map Interaction Contract

### States

| State | Trigger | Visual |
|-------|---------|--------|
| Default | App load | All countries rendered; has-photos countries filled blue (opacity 0.45); no-photos countries filled grey (opacity 0.2) |
| Hover | `mouseover` event on polygon | fillOpacity increases to 0.65; border weight 2px; border color darkens |
| Selected | `click` event on polygon | fillOpacity 0.75; border weight 3px; `--color-accent-dark` border; country panel opens |
| Selected + hover | cursor over selected country | Selected styles remain dominant (no override on hover of selected) |
| Deselect | Click same country again OR close panel | Returns to default/has-photos state; panel closes |

### Zoom / Pan

- Initial center: `[20, 0]` (equatorial center, shows most continents).
- Initial zoom: `2` (fits most of the world on a standard 1280px viewport).
- Min zoom: `2` (prevent over-zooming out to empty space).
- Max zoom: `6` (country-level; city-level zoom is Phase 3 pin detail).
- `worldCopyJump: true` on `<MapContainer>` — prevents jarring wraps when panning east/west.
- Scroll to zoom: enabled (Leaflet default).
- Double-click zoom: enabled (Leaflet default).

### Tile Provider

- Default tile URL (env-configurable via `VITE_TILE_URL`):
  `https://tiles.stadiamaps.com/tiles/alidade_smooth/{z}/{x}/{y}{r}.png`
- Attribution: `&copy; <a href="https://stadiamaps.com/">Stadia Maps</a> &copy; <a href="https://openmaptiles.org/">OpenMapTiles</a> &copy; <a href="https://www.openstreetmap.org/copyright">OpenStreetMap</a>`
- `maxZoom: 18` on `<TileLayer>` (Stadia supports up to 20).

## Copywriting Contract

| Element | Copy | Notes |
|---------|------|-------|
| Upload CTA (idle) | "Add Photos" | Primary button in upload zone |
| Upload CTA (uploading) | "Uploading..." | Button disabled during upload |
| Upload CTA (drag-active) | "Drop to Add" | Shown as zone label during active drag |
| Upload success toast | "{N} photo{s} added to {Country Name}" | Inline, below upload zone; disappears after 4 s |
| Upload error — invalid type | "File not accepted. Use JPEG, PNG, WebP, or HEIC." | Below upload zone, `--color-destructive` text |
| Upload error — file too large | "File exceeds 25 MB limit. Please reduce the file size." | Below upload zone, `--color-destructive` text |
| Upload error — server failure | "Upload failed. Check your connection and try again." | Below upload zone, `--color-destructive` text |
| Gallery empty state heading | "No photos yet" | Shown when country has 0 photos; `--text-heading` |
| Gallery empty state body | "Click \"Add Photos\" to start your {Country Name} collection." | `--text-body`, `--color-text-muted` |
| First-run empty state (whole map) | "Your travel map is empty" (heading) + "Click any country to start adding photos." (body) | Centered overlay on the map, only shown on very first load before any uploads |
| Panel close button | "Close" | Text button, top-right of panel header |
| Accepted formats hint | "JPEG, PNG, WebP, HEIC — up to 25 MB each" | Static text below the upload button |

**Destructive actions in Phase 1:** None. Photo deletion is deferred to Phase 4 (POL-04). No destructive confirmation patterns required in this phase.

**Tone:** Minimal, direct. No exclamation marks. No marketing language. Country names are always shown as received from GeoJSON `properties.NAME` (title-cased natural language).

## Accessibility Contract (Phase 1 minimum)

- All interactive elements (upload button, thumbnails, close button, country polygons) must be keyboard-reachable.
- Leaflet country polygons: set `tabIndex` via `onEachFeature` so keyboard users can Tab through countries; Enter/Space triggers the click handler. (Leaflet does not do this by default — must be added explicitly.)
- Upload zone: `<label>` wrapping the hidden file input; `aria-label="Upload photos for {country name}"`.
- Gallery thumbnails: `alt` attribute set to `photo.originalFilename` (from API).
- Lightbox (`yet-another-react-lightbox`): built-in ARIA roles and focus trap — do not override.
- Color contrast: `--color-text` (`#111827`) on `--color-surface` (`#ffffff`) = 17.7:1 (WCAG AA passes).
- Color contrast: `--color-text-muted` (`#6b7280`) on `--color-surface` (`#ffffff`) = 4.6:1 (WCAG AA passes for large text; borderline for 14px — acceptable for secondary copy).
