# Phase 1 — UI Design Contract
## Country Map & Per-Country Photos

> Visual and interaction contract for Phase 1 frontend.
> Generated by gsd-ui-researcher. Consumed by gsd-ui-checker, gsd-planner, gsd-executor, gsd-ui-auditor.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | **Tailwind CSS** (utility-first) |
| Preset | Tailwind default theme, extended with PhotoMap design tokens |
| Component library | none — plain React components styled with Tailwind utility classes |
| Icon library | none in Phase 1 — use text labels for actions; Phase 4 may add Lucide |
| Font | system-ui stack (no web font download — keeps Phase 1 lean) |

**Rationale:** Styling uses **Tailwind CSS** (user decision). Tailwind integrates with Vite via the official `@tailwindcss/vite` plugin (Tailwind v4) — no separate PostCSS config needed. All PhotoMap design tokens below (spacing, type scale, colors) are declared **once** in the Tailwind theme (`@theme` block in `client/src/index.css` for Tailwind v4, or `tailwind.config.js` `theme.extend` for v3) and consumed as utility classes (e.g. `bg-accent`, `p-md`, `text-heading`). The Tailwind theme is the single source of truth for Phase 1 styling. The token tables below define the **design contract values**; the "Tailwind Configuration" section at the end maps each to its theme token + utility class.

> **Note on Leaflet:** Country-polygon styling (the Map layer color table below) is applied via JavaScript style objects passed to Leaflet — Tailwind utility classes do NOT apply to Leaflet's SVG/canvas-rendered polygons. Those hex/opacity values are set directly in `styleCountry()`, using the same palette values defined in the Tailwind theme.

**Font stack (body and UI):**
```css
font-family: system-ui, -apple-system, BlinkMacSystemFont, 'Segoe UI', Roboto, sans-serif;
```

## Spacing Scale

Declared values (all multiples of 4). These match Tailwind's default spacing scale, so **stock Tailwind utilities are used** — no custom spacing tokens required.

| Token | Value | Tailwind utility (e.g. padding) | Usage |
|-------|-------|---------------------------------|-------|
| xs | 4px | `p-1` / `gap-1` | Icon gaps, chip padding, tight inline spacing |
| sm | 8px | `p-2` / `gap-2` | Compact element spacing, input inner padding |
| md | 16px | `p-4` / `gap-4` | Default element spacing, card padding |
| lg | 24px | `p-6` / `gap-6` | Section padding, panel header padding |
| xl | 32px | `p-8` / `gap-8` | Layout gaps, gallery grid gap |
| 2xl | 48px | `p-12` | Major section breaks |
| 3xl | 64px | `p-16` | Page-level vertical rhythm (rarely used) |

Exceptions:
- **Touch targets on mobile:** minimum 44px height for all interactive elements (upload button, gallery thumbnails, close button). Not on the 4-scale — use `min-h-11` (44px) Tailwind utility where needed.
- **Map full-bleed:** `h-dvh w-full` (`height: 100dvh; width: 100%`).

## Typography

Four roles, two weights. Body at 400, labels and headings at 600. Defined as custom Tailwind `fontSize` tokens (each bundles size + line-height).

| Role | Size | Weight | Line Height | Tailwind class | Usage |
|------|------|--------|-------------|----------------|-------|
| Body | 16px | 400 | 1.5 | `text-body font-normal` | Gallery captions, upload instructions, error messages, general prose |
| Label | 14px | 600 | 1.4 | `text-label font-semibold` | Button text, photo count badge, form field labels, tab labels |
| Heading | 20px | 600 | 1.2 | `text-heading font-semibold` | Country name in panel header |
| Display | 28px | 600 | 1.1 | `text-display font-semibold` | First-run empty state headline only |

**Tailwind `fontSize` tokens** (see Tailwind Configuration section):
```
body:    ['16px', { lineHeight: '1.5' }]
label:   ['14px', { lineHeight: '1.4' }]
heading: ['20px', { lineHeight: '1.2' }]
display: ['28px', { lineHeight: '1.1' }]
```

**Rules:**
- No other font sizes are permitted in Phase 1 — use only `text-body`, `text-label`, `text-heading`, `text-display`.
- Do not introduce a third weight. Only `font-normal` (400) and `font-semibold` (600). Bold (700) and light (300) are out of scope.
- Truncate long country names in the panel header with the `truncate` utility (`text-overflow: ellipsis; white-space: nowrap; overflow: hidden`) rather than wrapping.

## Color

60/30/10 split. The map tile (Stadia `alidade_smooth`) is a neutral grey-beige — the UI chrome must not compete with the map.

Defined as custom Tailwind `colors` tokens; consumed as `bg-*` / `text-*` / `border-*` utilities.

| Role | Hex | Tailwind token (utility) | Usage |
|------|-----|--------------------------|-------|
| Dominant (60%) | `#f8f9fa` | `bg` (`bg-bg`) | App background behind the map, panel body background |
| Secondary (30%) | `#ffffff` | `surface` (`bg-surface`) | Country side-panel surface, upload dropzone background, lightbox toolbar background |
| Accent (10%) | `#3b82f6` | `accent` (`bg-accent`) | **Reserved: has-photos country polygon fill, upload CTA button, active selection ring** |
| Accent dark | `#1d4ed8` | `accent-dark` (`bg-accent-dark`) | Selected country polygon border (hover/selected state) |
| Accent subtle | `rgba(59,130,246,0.15)` | `accent-subtle` (`bg-accent-subtle`) | Upload dropzone drag-active highlight, photo count badge background |
| Neutral border | `#e5e7eb` | `border` (`border-border`) | Panel border, gallery thumbnail border, dropzone idle border |
| Text primary | `#111827` | `text` (`text-text`) | All body copy, headings, labels |
| Text muted | `#6b7280` | `text-muted` (`text-text-muted`) | Empty state body copy, file count secondary text |
| Destructive | `#dc2626` | `destructive` (`text-destructive`) | Not used in Phase 1 (no delete action). Reserved for Phase 4 delete confirmation. |
| Map overlay | `rgba(0,0,0,0.4)` | `overlay` | Lightbox backdrop (applied via CSS override, see Lightbox) |

**Accent reserved for:** has-photos country polygon fill, upload CTA button background, selected country polygon border color (`accent-dark`). Never used for body text, borders, or passive states.

**Map layer colors (Leaflet polygon styles — not CSS custom properties, set in JS):**

| State | fillColor | fillOpacity | color (border) | weight |
|-------|-----------|-------------|----------------|--------|
| Default — no photos | `#e5e7eb` | 0.2 | `#9ca3af` | 1 |
| Default — has photos | `#3b82f6` | 0.45 | `#9ca3af` | 1 |
| Hover (any country) | current fill | 0.65 | `#374151` | 2 |
| Selected | `#3b82f6` | 0.75 | `#1d4ed8` | 3 |
| Selected — no photos | `#6366f1` | 0.6 | `#1d4ed8` | 3 |

Note: These values must be used in the `styleCountry()` function and `onEachFeature` handlers in `WorldMap.jsx` / `CountryLayer.jsx`. They are part of the design contract, not discretionary implementation choices.

## Layout

### App Shell (desktop — viewport >= 768px)

```
┌─────────────────────────────────────────────────────────┐
│                    WorldMap (full-bleed)                 │
│                  height: 100dvh                         │
│                  width: 100vw                           │
│                                                         │
│                                         ┌─────────────┐ │
│                                         │  Country    │ │
│                                         │  Panel      │ │
│                                         │  width:360px│ │
│                                         │  height:100%│ │
│                                         └─────────────┘ │
└─────────────────────────────────────────────────────────┘
```

- The map is always full-bleed (`position: fixed; inset: 0`).
- The country panel overlays the map on the right (`position: fixed; top: 0; right: 0; height: 100dvh; width: 360px; z-index: 400`).
- When no country is selected, the panel is absent (not rendered, not hidden).
- Leaflet's default z-index is 400 for the tile layer. The panel must use `z-index: 500` to sit above map controls.

### App Shell (mobile — viewport < 768px)

```
┌──────────────────────┐
│   WorldMap           │
│   height: 45dvh      │
│                      │
├──────────────────────┤
│   Country Panel      │
│   height: 55dvh      │
│   overflow-y: auto   │
└──────────────────────┘
```

- On mobile, the layout splits vertically: map top 45dvh, panel bottom 55dvh.
- The panel is a scrollable column (not a modal/drawer slide-in — simpler for Phase 1).
- The map portion remains interactive above the panel divider.
- Implementation: CSS media query at 768px breakpoint toggling the layout mode.

### Country Panel Internal Layout

```
┌──────────────────────────────┐
│ [Country Name]    [✕ Close]  │  header — --space-lg padding, --text-heading
├──────────────────────────────┤
│ [Upload Area]                │  dropzone — see Upload section
├──────────────────────────────┤
│ Photo grid (thumbnails)      │  gallery — see Gallery section
│                              │
└──────────────────────────────┘
```

---

> **Token-name convention in the tables below:** references written as `--color-accent`, `--space-lg`, `--text-heading` denote the corresponding Tailwind theme token / utility class defined above and in the Tailwind Configuration section (e.g. `--color-accent` → `bg-accent`/`accent`, `--space-lg` → `p-6`/`6`, `--text-heading` → `text-heading`). They describe the design contract value, not a literal CSS custom property — apply them as Tailwind utilities.

## Component Specifications

### Country Panel

| Property | Value |
|----------|-------|
| Width (desktop) | 360px fixed |
| Height | 100dvh (desktop) / 55dvh (mobile) |
| Background | `--color-surface` |
| Border | 1px solid `--color-border` on left edge (desktop) / top edge (mobile) |
| Box shadow | `0 0 24px rgba(0,0,0,0.12)` |
| Padding | `--space-lg` (header), `--space-md` (body) |
| Overflow | `hidden` on panel; `auto` on the gallery scroll region |
| z-index | 500 |
| Transition | none in Phase 1 — panel appears/disappears instantly |

**Panel header:**
- Country name: `--text-heading`, `--font-weight-semibold`, `--color-text`, single line with ellipsis.
- Close button: text "Close" at `--text-label`, `--color-text-muted`. Position: right edge of header row. Minimum size 44x44px touch target.

### Upload Zone (inside country panel)

| Property | Value |
|----------|-------|
| Layout | Full width of panel body, above gallery |
| Idle state | Dashed border 2px `--color-border`; background `--color-surface`; padding `--space-lg` |
| Drag-active state | Dashed border 2px `--color-accent`; background `--color-accent-subtle` |
| Height (idle) | 80px collapsed; expands to show file list on drag |
| Upload CTA button | See Copywriting section below |
| File input | Hidden `<input type="file" multiple accept="image/jpeg,image/png,image/webp,.heic,.heif">` |
| Accepted format hint | `--text-label`, `--color-text-muted`: "JPEG, PNG, WebP, HEIC — up to 25 MB each" |

**Upload button spec:**
- Background: `--color-accent` (`#3b82f6`)
- Text: `--color-surface` (`#ffffff`), `--font-weight-semibold`, `--text-label`
- Padding: `--space-sm` vertical, `--space-md` horizontal
- Border radius: 6px
- Hover: `--color-accent-dark` (`#1d4ed8`) background
- Min height: 44px (touch target)
- State — uploading: button disabled, text changes to "Uploading..." (see Copywriting)

### Photo Gallery Grid (inside country panel)

| Property | Value |
|----------|-------|
| Layout | CSS grid, `grid-template-columns: repeat(3, 1fr)` |
| Gap | `--space-xs` (4px) between thumbnails |
| Thumbnail size | 1:1 aspect ratio, fills grid column (`aspect-ratio: 1; object-fit: cover`) |
| Border radius | 4px on each thumbnail |
| Cursor | `pointer` on thumbnails |
| Hover | `opacity: 0.85` overlay |
| Scroll region | Panel body below upload zone; `overflow-y: auto` |
| Minimum thumbnail size | 80px (panel 360px - 2×16px padding - 2×4px gap) / 3 = ~105px — satisfies 44px min touch target |

**Photo count badge (on map country polygon):**
- Rendered as a Leaflet `DivIcon` positioned at the country centroid.
- Background: `--color-accent-subtle`; border: 1px solid `--color-accent`; text: `--color-accent`, `--text-label`, `--font-weight-semibold`.
- Shape: pill — `border-radius: 12px; padding: 2px 8px`.
- Show only when count >= 1. Do not render for countries with zero photos.
- Maximum displayed: if count > 99, show "99+".

### Lightbox

Library: `yet-another-react-lightbox` v3.32.0 (from RESEARCH.md).

| Property | Value |
|----------|-------|
| Backdrop | `--color-overlay` (`rgba(0,0,0,0.4)`) — library default is darker; override to 0.4 |
| Open trigger | Click on any thumbnail in the gallery |
| Navigation | Keyboard arrow keys + on-screen prev/next chevrons (library default) |
| Close | Escape key + click outside + "✕" button (library default) |
| Zoom | Disabled in Phase 1 (defer to Phase 4 polish) |
| Thumbnails strip | Disabled in Phase 1 (gallery grid serves this role) |
| Slide count indicator | Enabled (library default "N / total") |

**YARL CSS override** (import after library CSS):
```css
.yarl__backdrop { background: rgba(0, 0, 0, 0.4); }
```

## Map Interaction Contract

### States

| State | Trigger | Visual |
|-------|---------|--------|
| Default | App load | All countries rendered; has-photos countries filled blue (opacity 0.45); no-photos countries filled grey (opacity 0.2) |
| Hover | `mouseover` event on polygon | fillOpacity increases to 0.65; border weight 2px; border color darkens |
| Selected | `click` event on polygon | fillOpacity 0.75; border weight 3px; `--color-accent-dark` border; country panel opens |
| Selected + hover | cursor over selected country | Selected styles remain dominant (no override on hover of selected) |
| Deselect | Click same country again OR close panel | Returns to default/has-photos state; panel closes |

### Zoom / Pan

- Initial center: `[20, 0]` (equatorial center, shows most continents).
- Initial zoom: `2` (fits most of the world on a standard 1280px viewport).
- Min zoom: `2` (prevent over-zooming out to empty space).
- Max zoom: `6` (country-level; city-level zoom is Phase 3 pin detail).
- `worldCopyJump: true` on `<MapContainer>` — prevents jarring wraps when panning east/west.
- Scroll to zoom: enabled (Leaflet default).
- Double-click zoom: enabled (Leaflet default).

### Tile Provider

- Default tile URL (env-configurable via `VITE_TILE_URL`):
  `https://tiles.stadiamaps.com/tiles/alidade_smooth/{z}/{x}/{y}{r}.png`
- Attribution: `&copy; <a href="https://stadiamaps.com/">Stadia Maps</a> &copy; <a href="https://openmaptiles.org/">OpenMapTiles</a> &copy; <a href="https://www.openstreetmap.org/copyright">OpenStreetMap</a>`
- `maxZoom: 18` on `<TileLayer>` (Stadia supports up to 20).

## Accessibility Contract (Phase 1 minimum)

- All interactive elements (upload button, thumbnails, close button, country polygons) must be keyboard-reachable.
- Leaflet country polygons: set `tabIndex` via `onEachFeature` so keyboard users can Tab through countries; Enter/Space triggers the click handler. (Leaflet does not do this by default — must be added explicitly.)
- Upload zone: `<label>` wrapping the hidden file input; `aria-label="Upload photos for {country name}"`.
- Gallery thumbnails: `alt` attribute set to `photo.originalFilename` (from API).
- Lightbox (`yet-another-react-lightbox`): built-in ARIA roles and focus trap — do not override.
- Color contrast: `--color-text` (`#111827`) on `--color-surface` (`#ffffff`) = 17.7:1 (WCAG AA passes).
- Color contrast: `--color-text-muted` (`#6b7280`) on `--color-surface` (`#ffffff`) = 4.6:1 (WCAG AA passes for large text; borderline for 14px — acceptable for secondary copy).

## Tailwind Configuration

Setup: install Tailwind and the Vite plugin, register the plugin in `client/vite.config.js`, and import Tailwind in `client/src/index.css` (imported once in `main.jsx`). This spec targets **Tailwind v4** (CSS-first `@theme`); if the planner pins **v3**, use the equivalent `tailwind.config.js` `theme.extend` shown below.

**Tailwind v4 — `client/src/index.css`:**
```css
@import "tailwindcss";

@theme {
  /* Colors (spacing uses Tailwind's default 4px scale — no override needed) */
  --color-bg:            #f8f9fa;
  --color-surface:       #ffffff;
  --color-accent:        #3b82f6;
  --color-accent-dark:   #1d4ed8;
  --color-accent-subtle: rgba(59, 130, 246, 0.15);
  --color-border:        #e5e7eb;
  --color-text:          #111827;
  --color-text-muted:    #6b7280;
  --color-destructive:   #dc2626;
  --color-overlay:       rgba(0, 0, 0, 0.4);

  /* Type scale (size + line-height) */
  --text-body:    16px;   --text-body--line-height:    1.5;
  --text-label:   14px;   --text-label--line-height:   1.4;
  --text-heading: 20px;   --text-heading--line-height: 1.2;
  --text-display: 28px;   --text-display--line-height: 1.1;

  /* Font stack */
  --font-sans: system-ui, -apple-system, BlinkMacSystemFont, 'Segoe UI', Roboto, sans-serif;
}
```
Tailwind v4 auto-generates utilities from `@theme`: `bg-accent`, `text-text-muted`, `border-border`, `text-heading`, etc. The default spacing scale already provides `p-1/2/4/6/8/12/16` = 4/8/16/24/32/48/64px.

**Tailwind v3 alternative — `client/tailwind.config.js` `theme.extend`:**
```js
extend: {
  colors: {
    bg: '#f8f9fa', surface: '#ffffff',
    accent: { DEFAULT: '#3b82f6', dark: '#1d4ed8', subtle: 'rgba(59,130,246,0.15)' },
    border: '#e5e7eb',
    text: { DEFAULT: '#111827', muted: '#6b7280' },
    destructive: '#dc2626', overlay: 'rgba(0,0,0,0.4)',
  },
  fontSize: {
    body:    ['16px', { lineHeight: '1.5' }],
    label:   ['14px', { lineHeight: '1.4' }],
    heading: ['20px', { lineHeight: '1.2' }],
    display: ['28px', { lineHeight: '1.1' }],
  },
  fontFamily: { sans: ['system-ui','-apple-system','BlinkMacSystemFont','Segoe UI','Roboto','sans-serif'] },
}
```

**Layout constants** (not Tailwind tokens — use arbitrary values where needed): country panel desktop width `w-[360px]`; mobile breakpoint `md:` (Tailwind default 768px) drives the split layout.

**Leaflet polygon palette** (set in JS `styleCountry()`, NOT via Tailwind): reuse the same hex values — `#3b82f6` (accent), `#1d4ed8` (accent-dark), `#e5e7eb` (border), `#9ca3af`, `#374151`, `#6366f1` per the Map layer color table above.
